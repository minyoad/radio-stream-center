# Radio Channel & Source Manager (Radio 渠道与直播源管理系统)

这是一个功能齐全、现代化且极其轻量的 Radio 渠道和直播源管理系统。采用前后端全栈架构设计，提供包括一键并发测速、M3U/TXT 自动导入、GitHub 定时同步、EPG 频道元数据匹配在内的多种实用工具，并带有自定义播放接口与 Web 编辑导出功能。

---

## 🚀 核心特性

- 🖥️ **单页扁平化设计**：精美的卡片化布局，精心设计的负空间，全自适应适配各类桌面端分辨率。
- ⚡ **一键高并发测速**：基于 Express + 异步轮询的后台测速引擎，在极短时间内对所有源进行并发可用性测速。
- 📁 **自由分组及保护**：内置内置备用、内置分类保护，防误删机制完备，并提供安全的网页交互模态确认框。
- 📂 **M3U 与 TXT 双模导入**：完美兼容标准的 M3U 订阅或以文本格式贴入的 TXT 直播源。
- 🔄 **自动同步拉取**：支持添加 GitHub、第三方订阅源的自动定时抓取同步任务。
- 🛡️ **安全的后台安全架构**：屏蔽不安全的 client-side 直接网络交互与 CORS 错误，API 均由本地服务端进行高安全代理。
- 💾 **自动持久化与零负载设计**：采用本地文件持久化引擎（存储于 `./data/radio_data.json`），免去配置复杂数据库的烦恼。

---

## 🛠️ 技术栈说明

- **前端**：React 19 + Vite 6 + Tailwind CSS v4 + Lucide 矢量图标
- **后端**：Node.js 20 + Express 4 + esbuild（自适应编译生产单文件 CJS 捆绑包）
- **打包部署**：Docker / Docker Compose

---

## 📦 部署与运行方法

本系统支持 **本地手部署** 与 **Docker 一键部署**。

---

### 方法一：使用 Docker Compose（推荐，零环境污染）

系统已完整配置有 `Dockerfile` 与 `docker-compose.yml`，适合快速、稳健地在 Linux 服务器、群晖（Synology）等 NAS 设备中容器化运行。

#### 1. 前置要求
- 确保宿主机上已安装 `Docker` 和 `Docker Compose`。

#### 2. 一键启动
在项目根目录下，直接执行以下命令：

```bash
docker compose up -d
```

这会自动：
1. 采用 Docker **多阶段构建** 编译前端静态文件。
2. 采用 `esbuild` 自动把后端 TypeScript 编译整合，精简打包发布。
3. 开启容器自启保障（`restart: unless-stopped`）。
4. 将本机的 `./data` 文件夹挂载到容器中，确保频道、分组以及同步任务数据**永不丢失**。

#### 3. 访问与配置
- 运行成功后，通过浏览器访问：`http://localhost:3000` (或 `http://服务器IP:3000`)
- **端口自定义**：如需替换宿主机端口，只需编辑 `docker-compose.yml` 中的 `- "3000:3000"`，将其中的第一个 `3000` 更改为您想要的端口即可。
- **数据持久化位置**：本地生成的 `./data` 目录会包含 `radio_data.json`。如需迁移或备份数据，只需备份该文件夹即可。

---

### 方法二：本地开发与手动部署

如果您想在本地运行或参与本系统后续的定制开发，请按如下步骤配置：

#### 1. 前置要求
- 需要安装 `Node.js 20.x` 或以上版本。

#### 2. 安装项目依赖
```bash
npm install
```

#### 3. 启动开发服务器 (HMR Dev Mode)
```bash
npm run dev
```
启动后会在本地运行全栈自适应服务器，可直接通过面板在开发环境中编辑频道与调试测速。

#### 4. 本地编译与生产运行
当需要正式运行且不使用 Docker 时，可以使用生产版本编译：

```bash
# 1. 执行编译构建（打包前端至 dist/，并捆绑后端 ts 文件至 dist/server.cjs）
npm run build

# 2. 运行生产状态下的打包程序
npm run start
```

## 📝 数据结构 & 目录持久化

所有管理的直播源和配置都离线保存在容器或目录下的文件：
- 路径：`./data/radio_data.json`

当使用 Docker Compose 部署时，本地 `./data` 目录是自动生成的，并且具有极其安全的物理写入持久化保障，可以安全地通过定期备份此文件来备份整个 Radio 系统。

## 💡 使用指南

1. **新建分组与频道**：可通过右上角展开快捷操作，自定义国家、区域或者频道，如 "CCTV", "卫视频道" 等。
2. **导入线路**：一键添加多个视频流（M3U，HLS，FLV，RTSP等各种形式的直播流）。
3. **连通性测速**：点击仪表盘上的 "一键检测全部失效链接" 或 "并发测速"，系统会启动超快速异步测速流，并渲染出测速日志、最大可用延时与活跃线路条数。
4. **一键过滤**：对测速失败的线路可使用 "一键清理失效线路" 进行清理，极大减少冗余无用播放源。
